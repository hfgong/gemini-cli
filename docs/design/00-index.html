Phase 1: Prompt & Context Tuning IMPLEMENTED

Prompt wording and compression parameter changes. Low-risk, low-effort, limited impact ceiling.

01 — Prompt & Context Changes (A through I)
9 changes to system prompts, compression threshold, and context management. Flip directive/inquiry default, soften confirmations, reduce narration, increase persistence.
Done

Phase 2: Structural Agent Improvements IMPLEMENTED

Architecture-level changes to the agent loop and verification strategy. Now defaulted ON in CLI flags.

02 — Test-Driven Verification
Three flag-guarded features: --verify-before-complete (completion gate requiring tests pass), --protect-tests (prefer fixing source over tests), --reproduce-first (write reproduction script before fixing). All default ON.
Done
02b — Time Budget Awareness
--time-budget N flag injects time management guidance into system prompt. FeatureBench wrapper auto-injects from task timeout. Addresses 2/141 timeout failures.
Done
02c — API Crash Retry (FeatureBench wrapper)
Retry logic in gemini_cli_local.py: 3 retries with exponential backoff for zero-token API crashes. Smart detection distinguishes API crashes from real failures. Addresses 6/141 API failures.
Done
03 — Smart Context Management
Surgical context loading: extract only relevant symbols instead of whole files. Two-phase exploration (scan then deep-read). Address Gemini's context fragility.
Proposed
04 — Structured Exploration & Backtracking
Multi-hypothesis exploration with explicit backtracking. Failed approach tracking. Prevents the agent from tunneling into wrong solutions.
Proposed

Phase 3: New Capabilities FUTURE

New tools and agent capabilities. Requires deeper integration work.

05 — Semantic Code Search (planned)
AST-aware search beyond grep/glob. Find callers, implementations, type hierarchies. Reduce irrelevant context from text-based search.
Future
06 — Multi-Agent Decomposition (planned)
Break complex tasks into sub-tasks handled by specialized lightweight agents. Keep each agent's context small and focused.
